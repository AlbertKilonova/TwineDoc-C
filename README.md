# Twine Doc-C 

这里是 **Twine Doc-C**！一个为了让大家都能开心写出文字游戏而诞生的小地方喵~ 

虽然 Twine 很有趣，但外语文档总是让人头大对吧？所以波（AlbertKilonova）和好朋友们决定把它们统统集合起来，做成中文的！

### 我们的宝藏

这里藏着好多厉害的故事格式说明书哦：
- **Harlowe**：默认的小可爱，最适合新手入门啦！
- **SugarCube**：功能超强大的方糖，大触们都爱它。
- **Snowman**：给喜欢玩 JavaScript 的小伙伴准备的雪人。
- **Chapbook**：优雅又现代的小册子。

### 本地开发与构建指南

请遵循以下步骤在本地环境中运行或构建本项目：

#### 1. 环境要求
- **Node.js**: v18.0.0 或更高版本
- **包管理器**: npm (随 Node.js 一同安装)

#### 2. 获取源码
克隆本仓库至本地目录：
```bash
git clone https://github.com/AlbertKilonova/TwineDoc-C.git
cd TwineDoc-C
```

#### 3. 安装依赖
执行以下命令安装项目所需的全部依赖项：
```bash
npm install
```

#### 4. 启动开发服务器
启动本地热重载开发服务器，默认访问地址为 `http://localhost:5173`：
```bash
npm run docs:dev
```

#### 5. 构建静态站点
生成的静态文件将存放在 `docs/.vitepress/dist` 目录下，用于生产环境部署：
```bash
npm run docs:build
```

### 🛠️ 贡献小技巧（必看喵！）

为了让文档排版整齐且网址优雅，波在项目里用了点小技巧喵，请大家贡献时遵守以下规范：

- **排序号码（数字前缀）**
  我们在文件夹和文件名前加了 `数字-` 前缀（如 `1-Harlowe`）。这是为了让侧边栏乖乖听话按顺序排列。请大家新建文件夹或文件时也记得带上数字序号喵！

- **路由（自动去数字）**
  波配置了 `rewrites` 规则。虽然文件物理路径叫 `1-Harlowe/index.md`，但浏览器里的实际地址是 `/Harlowe`。
  - **写链接时**：请**不要**在路径里带数字！直接写 `[Harlowe](/Harlowe)`。
  - **侧边栏**：插件会自动隐藏数字前缀，只显示漂亮的标题。

- **代码高亮与行号**
  项目开启了 `lineNumbers` 配置。编写代码块时请标明语言，这样左侧会自动生成行号，方便大家阅读。

- **清爽的网址**
  我们开启了 `cleanUrls` 模式。在文档内互相引用时，不需要加上 `.html` 后缀，直接指向路径即可。
